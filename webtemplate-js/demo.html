Healthcare Data Conversion: Flat ↔ Structured ↔ Raw openEHR JSON

Phase 1 Proof of Concept: This demo shows vanilla JavaScript conversion between healthcare data formats without external dependencies or frameworks. Features include real-time validation, bi-directional conversion, and openEHR canonical JSON generation.

Flat Format (Form Data)

vitals/temperature|magnitude = 37.5
vitals/temperature|unit = °C
ctx/language = en
ctx/setting = emergency care
ctx/composer_name = Dr. Johnson

Structured JSON Format

{
  "vitals": {
    "temperature": {
      "magnitude": 37.5,
      "unit": "°C"
    }
  },
  "_context": {
    "language": "en",
    "setting": "emergency care",
    "composer_name": "Dr. Johnson"
  }
}

Raw openEHR Canonical JSON

{
  "@type": "COMPOSITION",
  "archetype_node_id": "demo-vitals-v1",
  "name": {
    "@type": "DV_TEXT",
    "value": "Health Data"
  },
  "language": {
    "@type": "CODE_PHRASE",
    "terminology_id": {
      "@type": "TERMINOLOGY_ID",
      "value": "ISO_639-1"
    },
    "code_string": "en"
  },
  ...
}

Key Methods

  • convertFromFlatToStructured(flatData, context) - Convert form data to structured JSON
  • convertFromStructuredToFlat(structuredJson) - Convert structured JSON to flat format
  • convertToRawOpenEHR(flatData, context) - Convert to openEHR canonical format
  • validateFlatData(flatData) - Validate healthcare data with errors and warnings
  • fromJsonTemplate(templateJson) - Create instance from template definition
